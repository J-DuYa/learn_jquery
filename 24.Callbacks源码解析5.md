#### "unique"的设计

Ensures a callback can only be added once (so there are no duplicates in the list).
> 确保一个回调函数只被添加一次（回调列表中不应有重复的函数）

unique的使用

```javascript
// 在不使用unique时
var callbacks = $.Callbacks();
function fn1(val) {
    console.log("fn1 says:" + val);
}
function fn2(val) {
    console.log("fn2 says:" + val);
}

callbacks.add(fn1);
callbacks.fire("foo");
callbacks.add(fn1);
// 第二次调用fire时,list有两个相同的fn1函数[fn1, fn1],所以fn1被执行了两次
callbacks.fire("bar");

// 输出
// fn1 says:foo
// fn1 says:bar
// fn1 says:bar
```

```javascript
// 在使用unique时
var callbacks = $.Callbacks("unique");

callbacks.add(fn1);
callbacks.fire("foo");
// 此时fn1在回调列表中已经存在，所以没被加进去。
callbacks.add(fn1);
// 第二次调用fire时,list只有一个fn1函数[fn1],所以fn1被执行了一次
callbacks.fire("bar");

// 输出
// fn1 says:foo
// fn1 says:bar
```
